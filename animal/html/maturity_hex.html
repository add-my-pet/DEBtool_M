maturity_hex

Gets maturity as function of length for model hex

Contents

  • Syntax
  • Description
  • Remarks
  • Example of use

Syntax

[H, a, info] = maturity_hex (L, f, p, lb, lj, le)

Description

Life history events b (birth), j (pupation), e (emergence); puberty coincides with birth Calculates the scaled maturity UH = MH/ {JEAm} at constant food density in the case of acceleration between b and j, pupa between j and e, (non-growing) imago after e, with UHb < UHj. Lengths must be ordered in time (so a single stepdown from L_j to 0 at j. Lengths are in the pupal stage if an earlier length was larger. So [.2 .3 .8] are lengths in the egg or larva stage (before pupation, because they only increase), but the last 3 in [.3 .2 .3 .8] are in the pupal stage, because they follow a larger length.

Input

  • L: n-vector with length
  • f: scalar with (constant) scaled functional response
  • p: 9-vector with parameters: kap kapV kapR g kJ kM v UHb UHe
  • lb: scaled length at birth at f
  • lj: scaled length at pupation at f
  • le: scaled length at emergence at f
  • tj: scalar with scaled age at pupation for f

Output

  • H: n-vector with scaled maturities: H = M_H/ {J_EAm} = E_H/ {p_Am}
  • info: scalar for 1 for success, 0 otherwise

Remarks

Maturity in the larval stage (between birth and pupation) stays at birth level. Maturity and structure are reset to zero at pupation and freeze are emergence of the imago. See maturity in absence of acceleration and maturity_j with accleration, maturity_s if accleration is delayed. Called from scaled_power_hex.

Example of use

[H, a, info] = maturity_hex(.4, 1, [.8,.95, .2, .002, .01, 0, .02, .2, .4, 2])
